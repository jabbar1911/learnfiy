Learn HTML – From Basics to Real Projects

Build web pages with HTML, create layouts, embed media, and prepare for frontend development.

Course Introduction

This Learnify HTML course teaches you how to structure web pages using HTML5. Start from tags and attributes, then learn semantic HTML, forms, media, and best practices. Perfect for beginners who want to build websites or move into frontend development.

Why Learn HTML?

  • Essential Web Skill — HTML is the foundation of every website.
  • Fast Results — See changes instantly in the browser.
  • Pair with CSS & JS — Start building interactive, styled sites.
  • Career Entry — Frontend, web design, content roles.

What You'll Learn

  • HTML tags, attributes and document structure
  • Semantic HTML5 (header, nav, main, article, section)
  • Text formatting, links, images, lists
  • Tables, forms, inputs and validations
  • Embedding media: audio, video, iframes
  • Responsive basics: meta viewport, mobile-first
  • Accessibility basics (alt, roles, aria)
  • Project workflow: folder structure & assets
  • Deploy simple pages to GitHub Pages / Vercel
  • Build portfolio-ready HTML projects

Course Syllabus

What You Will Learn

  • What is HTML?
    HTML (HyperText Markup Language) is the standard language used to build the structure of a webpage. It defines how text, images, buttons, and layout appear in the browser.
  • HTML Document Structure:
    Every HTML page follows a strict structure:
    • <!DOCTYPE html> — Declares HTML5 version
    • <html> — Root container for all content
    • <head> — Contains metadata, title, CSS links, and scripts
    • <body> — All visible elements displayed on the webpage
  • Commonly Used Tags:
    Essential tags used to structure content:
    • Headings: <h1> to <h6>
    • Paragraph: <p>
    • Block container: <div>
    • Inline container: <span>
  • HTML Attributes:
    Attributes provide additional information to elements:
    • id — Unique identifier
    • class — Used for grouping & styling
    • style — Inline CSS
    • title — Tooltip displayed on hover
  • Void (Self-Closing) Elements:
    These elements do not have closing tags:
    • <br> — Line break
    • <hr> — Horizontal separator
    • <img> — Displays an image
    • <input> — Form input field
  • Semantic HTML Tags:
    Tags that give meaning and structure to a webpage:
    • <header> — Page heading/top section
    • <nav> — Navigation menu
    • <main> — Main content area
    • <footer> — Bottom section of the webpage
    These enhance SEO, accessibility, and clarity.
  • Block vs Inline Elements:
    • Block elements — Start a new line and take full width (<div>, <p>, <h1>)
    • Inline elements — Stay in the same line (<span>, <a>, <strong>)
  • How Browsers Read HTML (DOM):
    Browsers convert HTML into a tree-based structure called the DOM (Document Object Model), allowing JavaScript to access and modify content dynamically.
  • Metadata in HTML:
    Metadata appears inside the <head> tag:
    • <title> — The title shown in the browser tab
    • <meta> — Charset, viewport, keywords, SEO description
  • Creating a Simple Webpage Layout:
    A basic webpage layout contains:
    • Header
    • Navigation bar
    • Main content area
    • Footer

Example Code

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Page</title>
  </head>

  <body>
    <header>
      <h1>Hello World</h1>
    </header>

    <main>
      <p>This is my first HTML webpage.</p>
    </main>

    <footer>

    </footer>
  </body>
</html>

What You Will Learn

  • Headings:

    • <h1> — Main page heading
    • <h2> — Section heading

  • Paragraphs:
    Paragraphs display blocks of text and automatically add spacing. 
    <p>This is a paragraph.</p>
  • Anchor Tags (Links):
    Used to create hyperlinks to other pages or resources. 
    <a href="https://example.com">Visit</a>
    Common attributes:
    • href — Link destination
    • target="_blank" — Opens link in a new tab
    • title — Tooltip on hover
  • Unordered Lists (Bulleted):
    Used when order does not matter. 
    <ul>
    <li>Item 1</li>
    <li>Item 2</li>
</ul>
  • Ordered Lists (Numbered):
    Used when order is important. 
    <ol>
    <li>Step 1</li>
    <li>Step 2</li>
</ol>
  • Nested Lists:
    Lists inside lists to create hierarchy. 
    <ul>
    <li>Fruits
        <ul>
            <li>Apple</li>
            <li>Mango</li>
        </ul>
    </li>
</ul>

    You can freely mix these elements.

<ul>

    <li>YouTube</li>

    • Use only one <h1> per page
    • Use descriptive link text (avoid "click here")

What You Will Learn

  • <img> Tag (Images):
    Used to display images. It is a self-closing (void) element.
    • src — Image file path
    • alt — Text shown if image fails to load
    • width — Controls image width
    • height — Controls image height
    <img src="logo.png" alt="Logo" width="200">
  • Responsive Images (srcset):
    Allows the browser to pick the best image size based on screen width. 
    <img 
    src="image-small.jpg"
    srcset="image-small.jpg 480w, image-medium.jpg 768w, image-large.jpg 1200w"
    alt="Responsive Image">
  • Lazy Loading Images:
    Improves performance by loading images only when they appear on screen. 
    <img src="banner.png" alt="Banner" loading="lazy">
  • Embedding Videos with <video>:
    The <video> tag is used to show videos on the webpage.
    • controls — Show video controls
    • autoplay — Play automatically
    • loop — Repeat video
    • muted — Mute video
    • poster — Custom thumbnail image
    <video controls width="400">
    <source src="intro.mp4" type="video/mp4">
</video>
  • Embedding Audio with <audio>:
    The <audio> tag is used to embed music or audio clips. 
    <audio controls>
    <source src="music.mp3" type="audio/mpeg">
</audio>
  • Combining Images & Media:
    You can mix images, videos, and audio in the same page easily. 
    <img src="logo.png" alt="Logo">

<video controls width="300">
    <source src="intro.mp4" type="video/mp4">
</video>

<audio controls>
    <source src="bgmusic.mp3" type="audio/mpeg">
</audio>
  • Best Practices:
    • Always use an alt attribute for accessibility
    • Use srcset for responsive images
    • Use loading="lazy" to improve page speed
    • Use optimized formats (JPG, PNG, SVG, MP4, MP3)
    • Keep file sizes small for faster loading

    <input type="text">

  • Labels:
    Labels improve accessibility and identify input fields.

    • Email format check

What You Will Learn

  • What is Responsive Web Design?
    Responsive design ensures your website automatically adapts to mobile, tablet, laptop, and desktop screens for better usability and readability.
  • Mobile-First Approach:
    The mobile-first workflow means designing for small screens first, then adding styles for larger devices. 
    p {
    font-size: 16px; /* Mobile */
}

@media(min-width: 768px) {
    p { font-size: 18px; } /* Tablet */
}

@media(min-width: 992px) {
    p { font-size: 20px; } /* Desktop */
}
  • Meta Viewport Tag:
    This tag ensures the website scales correctly on mobile devices. 
    <meta name="viewport" content="width=device-width, initial-scale=1">
    Without this tag, the website may appear zoomed out or broken on smartphones.
  • Responsive Design with Bootstrap:
    Bootstrap provides built-in responsive utilities and layouts:
    • Grid system (e.g., col-, col-md-, col-lg-)
    • Responsive spacing (mt-3 mt-md-0)
    • Responsive images (img-fluid)
    <div class="row">
    <div class="col-12 col-md-6">Left</div>
    <div class="col-12 col-md-6">Right</div>
</div>
  • Simple Responsive Techniques (Without Bootstrap):
    Responsive images: 
    img {
    max-width: 100%;
    height: auto;
}
    Fluid containers: 
    .container {
    width: 100%;
    padding: 10px;
}
    Flexbox layout: 
    .box {
    display: flex;
    flex-direction: column;
}

@media(min-width: 768px) {
    .box {
        flex-direction: row;
    }
}
    Using percentages instead of pixels: 
    .card {
    width: 100%;
}

@media(min-width: 768px) {
    .card {
        width: 50%;
    }
}
  • Full Responsive HTML Example: 
    <!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Responsive Demo</title>
</head>

<body>

    <h1>Welcome</h1>
    <p>This page adjusts to your screen size.</p>

    <div class="container">
        <div class="box">Box 1</div>
        <div class="box">Box 2</div>
    </div>

</body>
</html>
  • Best Practices:
    • Always include the viewport meta tag
    • Start with mobile-first CSS
    • Use flexible units (%, rem, vw, vh)
    • Use responsive images
    • Use Bootstrap for fast, responsive layouts
    • Test your website on different devices

What You Will Learn

  • Why Project Structure Matters:
    A clean folder structure keeps the project organized, easier to maintain, and scalable.
  • Common Project Folder Structure:
    A professional HTML project usually looks like this: 
    project/
│
├── index.html
├── css/
│     └── style.css
├── js/
│     └── script.js
├── img/
│     └── banner.png
└── assets/
      └── fonts/
    Folder meaning:
    • css/ — All styling files
    • js/ — All JavaScript files
    • img/ — Images, icons, backgrounds
    • assets/ — Fonts, media, or extra files
  • Linking CSS Files:
    Adding external CSS to HTML: 
    <link rel="stylesheet" href="css/style.css">
    This keeps HTML clean and makes styles reusable.
  • Linking JavaScript Files:
    JavaScript is usually linked before the </body> tag to improve performance. 
    <script src="js/script.js"></script>
  • Relative vs Absolute Paths:
    Relative paths (inside your project): 
    <img src="img/logo.png" alt="Logo">
<link rel="stylesheet" href="css/style.css">
    Absolute paths (full URL): 
    <img src="https://example.com/logo.png">
<script src="https://cdn.example.com/app.js"></script>
  • Types of Relative Paths:
    • Same folder: 
      <script src="app.js"></script>
    • Inside a child folder: 
      <link rel="stylesheet" href="css/style.css">
    • Go up one folder: 
      <img src="../img/pic.png">
  • Best Practices:
    • Keep HTML, CSS, JS in separate folders
    • Use clean file names (no spaces)
    • Store images inside img/
    • Keep reusable assets in the assets/ folder
    • Use relative paths for internal files
    • Use absolute paths for CDN links
  • Example of a Well-Organized Project: 
    myWebsite/
│
├── index.html
├── about.html
│
├── css/
│     ├── style.css
│     └── responsive.css
│
├── js/
│     ├── main.js
│     └── validation.js
│
├── img/
│     ├── logo.png
│     └── header-bg.jpg
│
└── assets/
      ├── fonts/
      └── icons/

What You Will Learn

  • Client-Side Validation:
    Validation checks user input before submitting to the server. It improves user experience, reduces errors, and prevents invalid data.
  • Required Attribute:
    Makes a form field mandatory. 
    <input type="text" required>
  • Pattern Attribute:
    Validates input using Regular Expressions (RegEx). Only alphabets: 
    <input type="text" pattern="[A-Za-z]+" required>
    Phone number (10 digits): 
    <input type="text" pattern="\d{10}" required>
    Password with number + 8 chars: 
    <input type="password" pattern="(?=.*[0-9]).{8,}" required>
  • Other Validation Attributes:
    • minlength / maxlength
    • min / max
    • step — increments
    • email — email format validation
    • url — URL validation
    <input type="number" min="1" max="100">
  • Custom Validation Messages:
    Use title to show custom hints. 
    <input type="text" pattern="[A-Za-z]+" 
       title="Only alphabets allowed">
  • Constraint Validation API (JavaScript):
    Check entire form validity: 
    form.checkValidity();
    Check single input validity: 
    input.checkValidity();
    Show validation error messages: 
    input.reportValidity();
    Stop form submission if invalid: 
    form.addEventListener("submit", function (e) {
    if (!form.checkValidity()) {
        e.preventDefault();
        form.reportValidity();
    }
});
  • Full Advanced Form Example: 
    <form id="registerForm">

    <label>Name
        <input type="text" name="name" required minlength="3">
    </label>

    <label>Email
        <input type="email" name="email" required>
    </label>

    <label>Password
        <input type="password" name="password" 
               pattern="(?=.*[0-9]).{8,}"
               title="Must be 8+ chars & include a number"
               required>
    </label>

    <label>Phone
        <input type="text" name="phone" pattern="\d{10}" required>
    </label>

    <button type="submit">Register</button>

</form>

<script>
const form = document.getElementById("registerForm");
form.addEventListener("submit", (e) => {
    if (!form.checkValidity()) {
        e.preventDefault();
        form.reportValidity();
    }
});
</script>
  • Best Practices:
    • Use HTML validation before JavaScript
    • Use required for important fields
    • Use pattern for strict input rules
    • Use title to guide users
    • Combine HTML + JS validation for full protection
    • Still validate on server — never rely only on client-side checks

What You Will Learn

  • What Is Deployment?
    Deployment means publishing your website online so anyone can view it using a URL. In this module, you will learn how to deploy using GitHub Pages, Netlify, and Vercel, including essential Git commands.
  • Deploying With GitHub Pages:
    GitHub Pages is perfect for static websites (HTML, CSS, JS). Steps:
    1. Create a GitHub account
    2. Create a new repository
    3. Upload your project (with index.html)
    4. Go to Settings → Pages
    5. Select Deploy from Branch
    6. Choose:
      • Branch: main
      • Folder: /root
    7. Save → GitHub generates your live link
    https://username.github.io/repository-name
  • Basic Git Commands: Initialize & upload project: 
    git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/username/repo.git
git push -u origin main
    Update project: 
    git add .
git commit -m "Updated website"
git push
  • Deploying With Netlify:
    Netlify allows extremely easy drag-and-drop hosting. Steps:
    1. Go to https://www.netlify.com
    2. Login using GitHub or Email
    3. Drag & drop your project folder
    4. Get instant live URL
    https://your-site-name.netlify.app
  • Deploying With Vercel:
    Vercel is great for frontend and JavaScript-based sites. Steps:
    1. Visit https://vercel.com
    2. Login with GitHub
    3. Import your repository
    4. Click "Deploy"
    https://project-name.vercel.app
  • Required Project Structure for Deployment: Hosting platforms expect an index.html in the root folder. 
    index.html
css/
js/
img/
  • Common Deployment Issues & Fixes:
    Issue Reason Fix
    Blank page Wrong file paths Use relative paths like ./css/style.css
    Images not loading Wrong folder name Check lowercase img/
    Git push fails No commit message git commit -m "msg"
    GitHub Pages not live Wrong branch selection Select main in Pages settings
  • Best Practices:
    • Always include index.html in project root
    • Use clean file names (no spaces)
    • Push updates via Git for auto-deployment
    • Use HTTPS links for assets
    • Check console errors before publishing

Image Example (Relative & Absolute Paths)
  • Relative Path (image inside img/ folder):
<img src="img/avatar.png" alt="User avatar">
  • Relative Path — go up one folder:
<img src="../img/avatar.png" alt="User avatar">
  • Absolute Path (full URL / CDN):
<img src="https://example.com/assets/img/avatar.png" alt="User avatar">

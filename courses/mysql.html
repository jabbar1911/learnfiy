Learn MySQL – From Basics to Real Databases

Master MySQL from foundational concepts to real-world database projects. Learn tables, queries, joins, functions, indexes, and build practical data-driven applications. Perfect for beginners preparing for backend development, data handling, and SQL problem-solving.

Course Introduction

This Learnify MySQL course teaches you MySQL from the absolute basics to practical, real-world database usage. Start with creating databases and tables, then move into writing queries, joins, functions, constraints, indexing, and managing relational data. Perfect for beginners who want to learn backend development, data handling, analytics, or prepare for SQL-based interviews and industry projects.

Why Learn MySQL?

  • Industry-Standard Database — MySQL is one of the most widely used relational databases in the world.
  • Essential for Backend Development — Powering websites, apps, CMS systems, and enterprise applications.
  • High Career Demand — SQL and MySQL skills are required in software development, data analysis, and IT roles.
  • Beginner-Friendly — Easy to understand SQL syntax makes it perfect for new learners.
  • Free & Open-Source — Learn and use MySQL without any cost, ideal for students and developers.
  • Works with Any Language — Integrates seamlessly with Python, Java, PHP, Node.js, and more.
  • Reliable & Fast — Trusted by companies worldwide for performance and stability.

What You'll Learn

  • MySQL Basics — Databases, tables, rows, columns, and SQL syntax structure.
  • SQL Queries — SELECT, INSERT, UPDATE, DELETE, and filtering data with WHERE.
  • Joins — INNER JOIN, LEFT JOIN, RIGHT JOIN, and combining data across tables.
  • Keys & Constraints — Primary keys, foreign keys, unique constraints, and NOT NULL.
  • Data Types — INT, VARCHAR, DATE, FLOAT, TEXT, and choosing the correct type.
  • Functions — COUNT, SUM, AVG, MIN, MAX, UCASE, LCASE, and string/date functions.
  • Ordering & Grouping — ORDER BY, GROUP BY, HAVING, and aggregated results.
  • Indexes — Creating and using indexes to improve query performance.
  • Subqueries — Nested SELECT queries to retrieve advanced filtered data.
  • Database Design — ER diagrams, normalization, and structuring relational data.
  • Views & Stored Procedures — Creating reusable virtual tables and server-side logic.
  • Transactions — COMMIT, ROLLBACK, ACID properties, and safe data updates.
  • Import & Export — Loading CSV/SQL files and backing up databases.
  • Mini Projects — Build small database systems and query-based applications.

Course Syllabus

  • What is a Database?
    A database is an organized collection of structured data that allows fast storage, retrieval, management, and updating of information. It is used in almost every application including banking, e-commerce, education, healthcare, and social media.
  • What is SQL?
    SQL (Structured Query Language) is the standard language used to communicate with relational databases. It helps perform tasks such as querying data, inserting records, updating information, deleting entries, and managing database structures.
  • Why MySQL?
    • Open-source & Free — No license fees, perfect for students & developers.
    • Highly Scalable — Supports small apps to enterprise-level systems.
    • Fast & Reliable — Optimized for high-performance read/write operations.
    • Widely Used — Trusted by Google, Facebook, Netflix, and many startups.
    • Cross-Platform — Works on Windows, macOS, and Linux.
    • Huge Community Support — Tons of tutorials, forums, and documentation.
  • Installing MySQL:
    • Download MySQL Community Server from the official website.
    • Install MySQL Workbench for a GUI-based SQL editor.
    • Set the root password during installation.
    • Configure the default port (3306) for MySQL Services.
    • Test the installation by connecting using Workbench or Terminal.
  • MySQL Tools You Will Use:
    • MySQL Server — Stores and processes databases.
    • MySQL Workbench — Visual tool for writing queries and designing schemas.
    • Command Line Client — Fastest way to execute SQL commands.
    • phpMyAdmin (optional) — Web-based interface for MySQL.
  • Connecting to MySQL (CLI):
    Use the command line to log into the MySQL server: 
    mysql -u root -p
    After entering the password, you can start writing SQL commands.

  • PRIMARY KEY
    A Primary Key uniquely identifies every record in a table.
    • Cannot contain NULL
    • Must be unique
    • A table can have only one PRIMARY KEY (can be composite) 
    emp_id INT PRIMARY KEY AUTO_INCREMENT;
  • FOREIGN KEY
    A Foreign Key creates a link between two tables and ensures data consistency.
    • Prevents inserting invalid department IDs
    • Used for one-to-many or many-to-many relationships 
    FOREIGN KEY (dept_id) REFERENCES departments(dept_id);
  • UNIQUE
    Ensures all values in the column are different. A table can have multiple UNIQUE constraints. 
    email VARCHAR(150) UNIQUE;
  • NOT NULL
    Ensures the column must always contain a value. Used for mandatory fields. 
    full_name VARCHAR(80) NOT NULL;
  • CHECK
    Validates data using a condition. Helps maintain clean data inside the database. 
    salary INT CHECK(salary > 10000);
  • DEFAULT
    Automatically assigns a value when the user does not provide one. 
    status VARCHAR(20) DEFAULT 'Active';
  • Composite Primary Key
    A primary key that uses more than one column. 
    PRIMARY KEY (student_id, course_id);
  • Adding Constraints (ALTER TABLE)
    ALTER TABLE employees
ADD CONSTRAINT fk_dept
FOREIGN KEY (dept_id)
REFERENCES departments(dept_id);
  • Dropping Constraints 
    ALTER TABLE employees DROP FOREIGN KEY fk_dept;
ALTER TABLE employees DROP INDEX email;
Constraint Description Example Use Case
PRIMARY KEY Uniquely identifies each record; cannot contain NULL. emp_id INT PRIMARY KEY Employee ID, Student Roll No.
FOREIGN KEY Links tables and maintains relational integrity. FOREIGN KEY(dept_id) REFERENCES dept(dept_id) Employee → Department relation
UNIQUE Ensures all column values are unique. email VARCHAR(150) UNIQUE Email, phone, Aadhaar number
NOT NULL Prevents storing empty (NULL) values. name VARCHAR(50) NOT NULL Required fields like name, password
CHECK Validates data with a given rule. age INT CHECK(age >= 18) Age restrictions, salary limits
DEFAULT Assigns a default value automatically. status VARCHAR(10) DEFAULT 'Active' User accounts, order statuses
Composite Key Two or more columns uniquely identify a row. PRIMARY KEY (student_id, course_id) Enrollment tables (many-to-many)
✔ Full Working Example: Table with All Constraints
CREATE TABLE employees (
    emp_id INT PRIMARY KEY AUTO_INCREMENT,
    full_name VARCHAR(80) NOT NULL,
    email VARCHAR(150) UNIQUE,
    age INT CHECK(age >= 18),
    salary INT DEFAULT 30000,
    dept_id INT,
    status VARCHAR(20) DEFAULT 'Active',
    FOREIGN KEY (dept_id) REFERENCES departments(dept_id)
);

  • GROUP BY Basics
    The GROUP BY clause groups rows that have the same values in a column and allows you to apply aggregate functions like COUNT(), AVG(), SUM(), MAX().
    Purpose Description Example Query
    Group by department Counts how many employees exist in each department SELECT department, COUNT(*) FROM employees GROUP BY department;
    Group by job role Shows total salary given to each role SELECT role, SUM(salary) FROM employees GROUP BY role; 
    SELECT department, COUNT(*)
FROM employees
GROUP BY department;
  • GROUP BY with Aggregate Functions
    When grouping, aggregate functions help summarize data for each group.
    Aggregate Description Example
    AVG() Average salary in each department SELECT department, AVG(salary) FROM employees GROUP BY department;
    MAX() / MIN() Highest & lowest salary per role SELECT role, MAX(salary), MIN(salary) FROM employees GROUP BY role;
    SUM() Total expenses per department SELECT department, SUM(salary) FROM employees GROUP BY department; 
    SELECT department, AVG(salary)
FROM employees
GROUP BY department;
  • HAVING Clause (Filter Groups)
    The HAVING clause filters the result of grouped data. WHERE filters rows BEFORE grouping, HAVING filters groups AFTER grouping.
    Use Case Description Example
    Departments with more than 5 employees Filters grouped data using COUNT() SELECT department, COUNT(*) FROM employees GROUP BY department HAVING COUNT(*) > 5;
    Roles with avg salary > 50000 Filters groups using AVG() SELECT role, AVG(salary) FROM employees GROUP BY role HAVING AVG(salary) > 50000; 
    SELECT department, COUNT(*)
FROM employees
GROUP BY department
HAVING COUNT(*) > 3;
  • Real-World GROUP BY Examples

  • Simple Subquery
    A simple subquery returns a single value and is used inside a WHERE clause. It runs first, and the result is used by the outer query.
    Type Description Example
    Scalar Subquery Returns exactly one value (SELECT MAX(salary) FROM employees) 
    SELECT * FROM employees
WHERE salary = (SELECT MAX(salary) FROM employees);
  • Subquery in FROM Clause (Inline View)
    A subquery can be treated as a temporary table by placing it inside the FROM clause. This is useful for applying operations on derived results.
    Purpose Description Example
    Inline Views Use subquery output as a table SELECT * FROM (SELECT salary FROM employees) AS temp; 
    SELECT emp_name, total_salary
FROM (
    SELECT name AS emp_name, salary * 12 AS total_salary
    FROM employees
) AS yearly_salaries
WHERE total_salary > 500000;
  • Subquery with IN
    A subquery can return multiple values, and the IN operator checks if a value matches any value from the subquery list.
    Operator Description Example Subquery
    IN Checks multiple returned values SELECT dept_id FROM departments WHERE location='Hyderabad' 
    SELECT name, department
FROM employees
WHERE department IN (
    SELECT dept_name FROM departments WHERE location = 'Hyderabad'
);
  • Subquery with EXISTS
    The EXISTS operator checks if the subquery returns at least one row. Commonly used in correlated subqueries. 
    SELECT name FROM employees e
WHERE EXISTS (
    SELECT * FROM salary_history s
    WHERE s.emp_id = e.emp_id
);
  • Correlated Subquery
    A correlated subquery depends on the outer query. It runs once for every row of the main query. 
    SELECT e.name, e.salary
FROM employees e
WHERE e.salary > (
    SELECT AVG(salary)
    FROM employees
    WHERE department = e.department
);
  • Real-World Subquery Examples

  • Stored Procedure
    A Stored Procedure is a saved block of SQL statements that you can execute whenever needed. Useful for automation, reducing repetition, and improving performance.
    Feature Description Example
    Reusable Write once, execute multiple times CALL getEmployees();
    Accept Parameters Pass values dynamically CALL filterSalary(50000);
    Improves Performance Stored and optimized by MySQL N/A

    SELECT name, department, salary FROM employees;
END $$

DELIMITER ;
  • Calling a Stored Procedure
    Execute the procedure using CALL. 
    CALL getEmployees();
  • Stored Procedure with Parameters
    Accepts inputs to filter results dynamically.

    WHERE department = dept;

SELECT @t AS Total_Employees;

  • START TRANSACTION
    Begins a new transaction. All statements after this point are treated as a single unit of work. 
    START TRANSACTION;
  • COMMIT
    Permanently saves all changes made in the current transaction. 
    COMMIT;
  • ROLLBACK
    Cancels the transaction and restores the database to the previous state. 
    ROLLBACK;
  • ACID Properties of Transactions
    Transactions in MySQL follow the ACID properties to ensure data reliability.
    Property Description
    Atomicity All steps in a transaction succeed or fail together.
    Consistency Ensures all database rules and constraints remain valid.
    Isolation Each transaction runs independently of others.
    Durability After COMMIT, data is permanently saved to the database.
  • Real-World Banking Example
    Transferring money between accounts requires all steps to succeed: 
    START TRANSACTION;

UPDATE accounts SET balance = balance - 500 WHERE account_no = 101;
UPDATE accounts SET balance = balance + 500 WHERE account_no = 202;

-- If both succeed
COMMIT;

-- If any error occurs
ROLLBACK;
  • Advanced Example (Insert + Update + Delete) 
    START TRANSACTION;

INSERT INTO orders (order_id, amount) VALUES (501, 2500);
UPDATE products SET stock = stock - 1 WHERE product_id = 10;
DELETE FROM cart WHERE user_id = 7 AND product_id = 10;

COMMIT;  -- Save all operations together

    If any query fails (for example, if stock is insufficient), use:

    ROLLBACK;
  • Benefits of Transactions
    • Protects data from incomplete or partial updates
    • Essential in banking, e-commerce, and ticket booking systems
    • Ensures stable and predictable database behavior
    • Maintains accuracy during multi-user access

    4. CRUD Operations
    -- Read all books
SELECT * FROM books;

-- Update book copies
UPDATE books SET available_copies = 4 WHERE book_id = 1;

-- Delete a member
DELETE FROM members WHERE member_id = 2;
    5. Join Example
    SELECT members.name, books.title, issue_records.issue_date
FROM issue_records
INNER JOIN members ON issue_records.member_id = members.member_id
INNER JOIN books ON issue_records.book_id = books.book_id;
    6. Create a View
    CREATE VIEW issued_books_view AS
SELECT members.name, books.title, issue_records.issue_date
FROM issue_records
JOIN members ON issue_records.member_id = members.member_id
JOIN books ON issue_records.book_id = books.book_id;
    7. Stored Procedure Example
    DELIMITER $$

CREATE PROCEDURE issueBook(IN b_id INT, IN m_id INT)
BEGIN
    INSERT INTO issue_records (book_id, member_id, issue_date)
    VALUES (b_id, m_id, CURDATE());

    UPDATE books SET available_copies = available_copies - 1
    WHERE book_id = b_id;
END $$

DELIMITER ;
    8. GROUP BY Report
    SELECT genre, COUNT(*) AS total_books
FROM books
GROUP BY genre;

Learn Flask – Build Dynamic Web Applications

Master Flask development from the basics to real-world web applications. Learn routing, templates, forms, databases, authentication, and API creation to build powerful web apps. Perfect for beginners preparing for web development, backend programming, and deployment.

Course Introduction

This Learnify Flask course guides you from the fundamentals of web development to building real-world web applications. Start with understanding the Flask framework, routing, and templates, then move into forms, databases, authentication, APIs, and deployment. Perfect for beginners who want to learn backend development, create dynamic websites, or build production-ready web applications using Python.

Why Learn Flask?

  • Simple & Lightweight — Flask provides a clean and minimal structure, making it easy for beginners to understand how web development works from the ground up.
  • Full Control Over Your Application — Unlike larger frameworks, Flask doesn’t force a specific project structure. You decide how your application should be organized, giving complete flexibility.
  • Perfect for APIs & Microservices — Flask is widely used for building REST APIs, backend services, and microservice-based systems due to its speed and minimal overhead.
  • Highly Extensible — Need authentication? Database ORM? Form validation? Flask has powerful extensions like Flask-SQLAlchemy, Flask-Login, Flask-WTF, and many more.
  • Fast Prototyping — Ideal for startups, hackathons, and student projects because you can turn ideas into working applications very quickly.
  • Strong Industry Usage — Companies use Flask for dashboards, AI/ML model deployment, admin panels, IoT apps, internal tools, and scalable backend services.
  • Great for Deploying Machine Learning Models — Flask is one of the most popular tools for serving ML models as APIs or web apps.
  • Active Community & Resources — Thousands of tutorials, videos, GitHub projects, and extensions help you learn and troubleshoot easily.

What You'll Learn

  • Introduction to Flask — Learn how Flask works internally, understand the request–response cycle, and set up your complete development environment with virtual environments and proper project structure.
  • Routing & URL Building — Create static and dynamic routes, pass parameters in URLs, and organize routes using Blueprints for scalable applications.
  • Templates & Jinja2 Engine — Build dynamic pages using Jinja2, template inheritance, loops, conditions, filters, and reusable components to create clean and maintainable UIs.
  • Handling Forms & User Input — Process HTML form data, validate inputs with Flask-WTF, manage GET/POST requests, and safely handle user-submitted information.
  • Working with Databases — Integrate Flask with SQLite/MySQL, design models, connect to databases, and perform full CRUD operations using both raw SQL and SQLAlchemy ORM.
  • User Authentication & Sessions — Implement user login, logout, session management, password hashing, access control, and protected routes using Flask-Login.
  • Building REST APIs — Create powerful APIs, return JSON data, accept JSON input, and structure API endpoints for real-world backend services.
  • Flask Extensions — Work with essential Flask extensions such as Flask-SQLAlchemy for databases, Flask-Migrate for migrations, Flask-WTF for forms, Flask-Login for authentication, and more.
  • Advanced Database Operations — Build relational models, define one-to-many and many-to-many relationships, apply migrations, write efficient queries, and maintain clean database schemas.
  • API Development & Integration — Build RESTful APIs, integrate them with JavaScript or mobile apps, and learn how to handle pagination, serialization, and error handling.
  • Application Deployment — Learn to deploy Flask applications on platforms like Render, Railway, Vercel (with adapter), PythonAnywhere, or even VPS servers using Gunicorn and Nginx.
  • Mini & Major Projects — Build multiple real-world apps including a blog system, a task manager, an authentication system, an API-based app, and a full-scale Flask project combining all learned concepts.

Flask Course Syllabus

  • What is Flask?
    Flask is a lightweight, modular, and powerful Python web framework that follows the micro-framework architecture. It allows you to build everything from simple web pages to full REST APIs, backend systems, dashboards, and microservices with minimal setup.

    Flask gives developers full control over project structure, making it ideal for learning, experimenting, and deploying real-world applications.
  • Why Flask?
    • Lightweight & extremely flexible
    • Perfect for beginners and professionals
    • Supports REST API development easily
    • Large ecosystem of extensions (SQLAlchemy, Flask-Login, Flask-WTF)
    • Widely used in ML model deployment & backend development
    • Simple to integrate with databases, templates, and front-end frameworks

  • What is Routing?
    Routing is the process of connecting a URL (such as /home or /login) to a specific function in your Flask application. Whenever a user visits a URL, Flask executes the function associated with that route.

    Routing forms the backbone of every Flask application — from simple websites to full API systems.
  • Basic Route Example
    This route displays text when the root URL (/) is visited. 
    @app.route("/")
def home():
    return "Welcome to Flask Routing!"

    Visit in browser:

    http://127.0.0.1:5000/
  • Dynamic Routes (Passing Values)
    Flask allows adding variables to URLs and reading them inside functions. 
    @app.route("/profile/<username>")
def profile(username):
    return f"User Profile: {username}"

    Example URL:

    http://127.0.0.1:5000/profile/Saketh
    Types of dynamic parameters:
    Type Usage Example Route
    string Default type (text) /user/<string:name>
    int Accepts only integers /product/<int:id>
    float Accepts decimal values /price/<float:value>
    path Accepts full paths /file/<path:filepath>

    Example using int:

    @app.route("/product/<int:pid>")
def product(pid):
    return f"Product ID: {pid}"
  • URL Building using url_for()
    url_for() helps generate URLs dynamically based on function names rather than hardcoding paths. 
    from flask import Flask, url_for

@app.route("/")
def home():
    return "Homepage"

@app.route("/contact")
def contact():
    return "Contact Page"

print(url_for("home"))      # / 
print(url_for("contact"))   # /contact

    Benefits of url_for():

    • Avoids hardcoding URLs
    • Automatically updates when routes change
    • Useful when passing parameters

    Example with parameters:

    url_for("profile", username="jabba")
  • HTTP Methods (GET, POST, PUT, DELETE)
    Flask routes can accept different HTTP request types.
    Method Purpose Example Usage
    GET Fetch data Loading a web page
    POST Send data to the server Form submissions
    PUT Update existing data Editing records via API
    DELETE Remove data Deleting users or posts
    Example Route with GET & POST: 
    @app.route("/login", methods=["GET", "POST"])
def login():
    if request.method == "POST":
        return "Form Submitted"
    return "Login Page"

    Example API with PUT & DELETE:

    @app.route("/update/<int:id>", methods=["PUT"])
def update(id):
    return f"Updated record {id}"

@app.route("/delete/<int:id>", methods=["DELETE"])
def delete(id):
    return f"Deleted record {id}"

  • What Are Templates?
    Templates are HTML files that allow you to display dynamic content using Flask’s template engine Jinja2. Instead of writing static HTML, you can inject variables, loops, conditions, and reusable layouts.

    Templates help separate:
    • Backend (Python logic)
    • Frontend (HTML layout)
    This makes your application cleaner and easier to maintain.
  • Rendering a Template
    The render_template() function loads HTML files from the templates/ folder. 
    from flask import Flask, render_template

app = Flask(__name__)

@app.route("/home")
def home():
    return render_template("home.html")

    Folder Structure:

    project/
│── app.py
│── templates/
    │── home.html
    │── profile.html
  • Jinja2 Syntax (Important Commands)
    Jinja2 lets you embed dynamic Python-like expressions inside HTML.
    Feature Syntax Description
    Variable {{ variable }} Prints Python variable inside HTML
    Condition {% if condition %} ... {% endif %} Conditional rendering
    Loop {% for item in list %} ... {% endfor %} Iterates through items
    Comments {# comment #} HTML comments inside Jinja2
    Filters {{ name|upper }} Modify output (uppercase, lower, title, length)

    Example:

    <h1>Welcome, {{ user }}</h1>

{% if age >= 18 %}
    <p>You are an adult.</p>
{% else %}
    <p>You are under 18.</p>
{% endif %}

<ul>
{% for fruit in fruits %}
    <li>{{ fruit }}</li>
{% endfor %}
</ul>
  • Passing Data to Templates
    You can send variables, lists, dictionaries, and objects from Flask to the HTML file.

    <h2>Name: {{ user.name }}</h2>

    home.html

    Template inheritance helps maintain a uniform layout across all pages, and makes your project easier to manage.

  • HTML Form Submission
    Flask handles form data sent from HTML using GET or POST methods. A basic HTML form inside a template: 
    <form action="/submit" method="POST">
    <input type="text" name="username" placeholder="Enter username">
    <button type="submit">Submit</button>
</form>

    Key Points:

    • Form must include name attributes
    • Default method is GET if not specified
    • POST method is used for sensitive data
  • Accessing Form Data in Flask
    Use Flask's request object to retrieve submitted values. 
    from flask import request

@app.route("/submit", methods=["POST"])
def submit():
    username = request.form["username"]
    return f"Hello, {username}"

    Alternative (safer) method using get():

    username = request.form.get("username")
  • GET vs POST (Important Differences)
    Method Description Use Case
    GET Data is visible in URL Searching, filtering, navigation
    POST Data is sent securely in the request body Login forms, uploads, sensitive input

    GET Example:

    @app.route("/search")
def search():
    keyword = request.args.get("q")
    return f"Results for {keyword}"

    POST Example:

    @app.route("/login", methods=["POST"])
def login():
    username = request.form.get("user")
    password = request.form.get("pass")
    return f"Welcome {username}"
  • Form Validation Basics
    Validation ensures the user enters correct and safe data before processing.

    Manual Validation Example:

    @app.route("/register", methods=["POST"])
def register():
    username = request.form.get("username")
    email = request.form.get("email")

    if not username or not email:
        return "All fields are required!"

    if "@" not in email:
        return "Invalid email format"

    return "Registration successful"

    Common Validation Rules:

    Validation Type Description Example
    Required Field must not be empty if not field:
    Email Format Must contain @ and domain "@" in email
    Length Check Minimum or maximum characters len(username) >= 3
    Number Validation Check if input is numeric age.isdigit()
  • Complete Flask Form Handling Example

    HTML Form:

    <form action="/feedback" method="POST">
    <input name="name" placeholder="Your Name">
    <textarea name="message"></textarea>
    <button type="submit">Send</button>
</form>

    Backend Code:

    @app.route("/feedback", methods=["POST"])
def feedback():
    name = request.form.get("name")
    message = request.form.get("message")

    if not name or not message:
        return "Please fill out all fields"

    return f"Thanks {name}, we received your message!"

  • What Are Cookies?
    Cookies are small pieces of data stored in the user's browser. They help identify users, save preferences, and track activity across requests.

    Common uses of cookies:

    • Remembering login info
    • Saving theme preferences (dark/light mode)
    • Tracking user sessions
    • Storing cart items for online shopping
    Property Description
    Name Key of the cookie
    Value Data stored (string only)
    Expiry How long the cookie stays in browser
    Path Which routes can access the cookie
  • Setting a Cookie in Flask
    Cookies are set using set_cookie() on the response object. 
    from flask import make_response

@app.route("/setcookie")
def setcookie():
    resp = make_response("Cookie has been set")
    resp.set_cookie("username", "Jabbar", max_age=3600)
    return resp

    Parameters of set_cookie():

    Parameter Description Example
    max_age Cookie expiry time (in seconds) max_age=3600
    path Accessible URL path path="/"
    httponly Protects cookie from JavaScript httponly=True
    secure Sends cookie only over HTTPS secure=True
  • Reading Cookies
    @app.route("/getcookie")
def getcookie():
    user = request.cookies.get("username")
    return f"Logged in as {user}"
  • What Are Sessions?
    Sessions store data on the server instead of the browser. Flask uses a secret key to encrypt session data and identify users.

    Why use sessions?

    • More secure than cookies
    • Can store complex data (dict, list)
    • Used for login systems and dashboards

    Example: Setting secret key

    app.secret_key = "mysecretkey"
    Feature Cookies Sessions
    Stored Location Client Browser Server
    Security Less secure More secure
    Data Type Only string Any Python object
    Use Case Themes, small preferences Authentication, dashboard data
  • Setting Session Data 
    from flask import session

@app.route("/login")
def login():
    session["user"] = "Jabbar"
    return "Session Set"

    Storing multiple values:

    session["cart"] = ["laptop", "mouse", "keyboard"]
  • Accessing Session Data 
    @app.route("/dashboard")
def dashboard():
    user = session.get("user")
    return f"Welcome, {user}"
  • Removing Session Data
    session.pop("user", None)

    Clear entire session:

    session.clear()
  • Complete Login Example Using Sessions 
    @app.route("/login", methods=["POST"])
def login():
    username = request.form.get("username")
    session["user"] = username
    return "Login Successful"

@app.route("/logout")

    SQL (Relational) SQLite, MySQL, PostgreSQL Structured, relationships, large applications

  • Setting Up SQLite Database
    SQLAlchemy integrates easily with SQLite (default for Flask development). 
    app.config["SQLALCHEMY_DATABASE_URI"] = "sqlite:///mydb.db"

    SQLite Advantages:

    • Lightweight & requires no server
    • Perfect for development and testing
    • Database stored in a single .db file
  • Creating a Database Model
    A model represents a table inside the database. 
    from flask_sqlalchemy import SQLAlchemy
db = SQLAlchemy(app)

class User(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(100))
    email = db.Column(db.String(120), unique=True)

    Model → Table Mapping

    Model Attribute DB Column Type Description
    id Integer (Primary Key) Unique ID for each user
    name String(100) Name of the user
    email String(120) Must be unique
    age Integer User's age

    Create all tables:

    with app.app_context():

</form>

    Important points:

    • Use POST for security
    • Each input must have a name attribute
    • Server receives data via request.form
  • Handling Signup Data
    Flask receives form data through the request object. 
    @app.route("/signup", methods=["POST"])
def signup():
    username = request.form.get("username")
    password = request.form.get("password")
    # Process registration

    Validation Checks:

    Check Example
    Empty fields if not username or not password
    Username length len(username) >= 3
    Password strength len(password) >= 6
    Check if user exists User.query.filter_by(username=username).first()
  • Password Hashing (Security)
    Passwords must **never** be stored as plain text. Flask uses Werkzeug's hashing functions. 
    from werkzeug.security import generate_password_hash

hashed_pw = generate_password_hash(password, method="sha256")

    Why hashing?

    • Prevents attackers from reading user passwords
    • Even developers can’t see original password
    • Hashing is one-way and irreversible
    Method Security Level
    sha256 Strong
    pbkdf2:sha256 Very strong (recommended)
  • Login Logic (Password Verification)
    When a user logs in, compare the stored hashed password with the entered password. 
    from werkzeug.security import check_password_hash

if check_password_hash(stored_password, password):
    session["user"] = username
    return redirect("/dashboard")
else:
    return "Invalid Credentials"

    Checklist before login succeeds:

    • User exists in database
    • Password matches hashed password
    • Session is created after login
  • Logout Functionality
    Logging out removes the session data. 
    @app.route("/logout")
def logout():
    session.pop("user", None)
    return redirect("/login")
  • Protecting Routes
    Some routes should only be accessible after login. 
    @app.route("/dashboard")
def dashboard():
    if "user" not in session:
        return redirect("/login")
    return "Welcome to Dashboard"

    Common Protected Routes:

    • Admin Panels
    • User Dashboards
    • Settings Pages
    • Order History
    Authentication Component Purpose
    Signup Create new account
    Login Verify credentials and create session
    Session Store info about logged-in user
    Logout Remove session and log out
    Route Protection Restrict access to logged-in users
  • Complete Authentication Flow Example 
    # SIGNUP
@app.route("/signup", methods=["POST"])
def signup():
    username = request.form["username"]
    password = request.form["password"]
    hashed = generate_password_hash(password)
    user = User(username=username, password=hashed)
    db.session.add(user)
    db.session.commit()
    return "User Registered"

# LOGIN
@app.route("/login", methods=["POST"])
def login():
    username = request.form["username"]
    password = request.form["password"]
    user = User.query.filter_by(username=username).first()

    if user and check_password_hash(user.password, password):
        session["user"] = username
        return "Logged In"
    return "Invalid Login"

# LOGOUT
@app.route("/logout")
def logout():
    session.pop("user", None)
    return "Logged Out"

  • What is an API?
    An API (Application Programming Interface) allows different applications to communicate with each other. In Flask, APIs mainly exchange data in JSON format using HTTP methods.

    Key Features of APIs:

    • Language independent
    • Lightweight and fast
    • Used in mobile apps, web apps, and microservices
    • Communicates using JSON, XML (JSON preferred)
  • Returning JSON Response
    Use jsonify() to return JSON from a Flask route. 
    from flask import jsonify

@app.route("/api/info")
def info():
    return jsonify({
        "app": "Learnify API",
        "version": 1.0
    })

    Response Format:

    Type Example Output
    JSON Object {"name": "Jabbar", "age": 21}
    JSON List [1, 2, 3]
  • Handling POST Request
    POST is used to send data to the server, usually in JSON format. 
    @app.route("/api/add-user", methods=["POST"])
def add_user():
    data = request.get_json()
    return jsonify({
        "message": "User received",
        "data": data
    })

    Example JSON Body:

    {
    "name": "Rahul",
    "age": 22
}
  • API Methods (HTTP Verbs)
    These define what operation the API performs.
    Method Purpose Example
    GET Retrieve data /api/users
    POST Create new data /api/users
    PUT Update existing data /api/users/10
    DELETE Remove data /api/users/10
  • Simple CRUD API Example
    Below is a mini CRUD API using Python list as temporary storage. 
    users = [
    {"id": 1, "name": "Jabbar"},
    {"id": 2, "name": "Amit"}
]

# GET - Fetch all users
@app.route("/api/users", methods=["GET"])
def get_users():
    return jsonify(users)

# POST - Add new user
@app.route("/api/users", methods=["POST"])
def create_user():
    data = request.get_json()
    users.append(data)
    return jsonify({"message": "User created", "data": data})

# PUT - Update user
@app.route("/api/users/<int:uid>", methods=["PUT"])
def update_user(uid):
    data = request.get_json()
    for u in users:
        if u["id"] == uid:
            u.update(data)
            return jsonify({"message": "User updated", "data": u})
    return jsonify({"error": "User not found"})

# DELETE - Remove user
@app.route("/api/users/<int:uid>", methods=["DELETE"])
def delete_user(uid):
    for u in users:
        if u["id"] == uid:
            users.remove(u)
            return jsonify({"message": "User deleted"})
    return jsonify({"error": "User not found"})

  • Why File Uploads in Flask?
    File uploads allow users to submit documents, images, reports, profile pictures, CSV files, etc. Flask provides a simple way to upload, validate, save, and download files securely.
  • Configuring Upload Folder
    You must set a folder where all uploaded files will be stored. 
    app.config["UPLOAD_FOLDER"] = "uploads/"

    Make sure the folder exists:

    project/
│── app.py
│── uploads/   ← uploaded files saved here
└── templates/
  • HTML File Upload Form
    Must include enctype="multipart/form-data" to send files. 
    <form method="POST" enctype="multipart/form-data">
    <input type="file" name="uploadfile">
    <button type="submit">Upload</button>
</form>
  • Accessing Uploaded File in Flask 
    file = request.files["uploadfile"]
print(file.filename)

    If no file was uploaded:

    if file.filename == "":
    return "No file selected"
  • Saving Uploaded File
    Flask allows you to store the uploaded file inside your upload directory. 
    file.save(os.path.join(app.config["UPLOAD_FOLDER"], file.filename))

    Full Example:

    @app.route("/upload", methods=["POST"])
def upload():
    file = request.files["uploadfile"]
    file.save(os.path.join(app.config["UPLOAD_FOLDER"], file.filename))
    return "File uploaded successfully!"
  • Allowed File Types
    Use this to restrict uploads to safe formats.
    Type Extensions
    Images png, jpg, jpeg
    Documents pdf, txt, docx
    Data Files csv, json 
    ALLOWED_EXT = {"png", "jpg", "pdf", "txt"}

    Checking file extension:

    def allowed(filename):
    ext = filename.rsplit(".", 1)[1].lower()
    return ext in ALLOWED_EXT
  • Secure File Upload Example (Validation Included) 
    @app.route("/upload", methods=["POST"])
def upload_file():
    file = request.files["uploadfile"]

    if file.filename == "":
        return "No file selected"

    if not allowed(file.filename):
        return "File type not allowed"

    filepath = os.path.join(app.config["UPLOAD_FOLDER"], file.filename)
    file.save(filepath)

    return f"File saved at {filepath}"
  • Downloading Files
    Use send_file() or send_from_directory(). 
    from flask import send_from_directory

@app.route("/download/<filename>")
def download(filename):
    return send_from_directory("uploads", filename)

    Example: Download data.csv

    /download/data.csv
  • Important Security Tips
    • Always validate file extensions
    • Never trust user file names
    • Consider renaming files before saving
    • Store uploaded files outside the project root in production

    Safe renaming example:

    import uuid

new_name = str(uuid.uuid4()) + ".png"
file.save(os.path.join(app.config["UPLOAD_FOLDER"], new_name))
